## 인증과 인가
- 인증: 사용자의 신원을 확인하는 과정
- 인가: 이미 인증된 사용자의 권한을 확인받는 과정

### 세션방식
세션에 인증정보를 저장하고 이를 이용해 인증을 처리하는 방식
- 장점: 세션 객체를 이용해, 사용자들의 상태를 관리할 수 있음 (동시접속 제한 등)
- 단점: 메모리를 과하게 사용, 데이터베이스를 자주 조회해야 함 => 서버에 부하가 일어날 수 있다

### 토큰방식(JWT)
토큰을 이용해 인증을 처리하는 방식
토큰에 나만 알 수 있는 서명을 넣어서 생성하기 때문에 요청이 왔을 때 서명을 사용하여 유효성을 검사할 수 있음

- 장점: 별도로 데이터베이스에 저장을 할 필요가 없어 데이터베이스에 접근하지 않아도 된다
- 단점: 
사용자들의 상태를 관리할 수 없음
토큰이 탈취되면 대처가 불가능하다

=> 액세스 토큰, 리프레쉬 토큰을 사용한다
액세스 토큰: 유효기간이 짧은 토큰 (30분~1시간)
리프레쉬 토큰: 유효기간이 긴 토큰 (2주일~1개월): 전송되는 경우가 적어 탈취를 당할 위험이 적다

리프레쉬 토큰을 이용해 액세스 토큰을 갱신할 수 있다
